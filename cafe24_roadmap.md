# Cafe24 상품 상세 옵션 커스터마이징 과제 - 로드맵

## 📋 과제 개요

- **목표**: 카페24 상품 상세페이지에 "골라담기" 형태의 커스텀 옵션 UI 구현
- **중점**: 디자인보다 **기능 안정성**과 **카페24 구조 이해** 우선
- **예상 소요 시간**: 약 8시간
- **제출 기한**: 2일 내 (테스트 안내 후)

---

## 🎯 핵심 요구사항 정리

### 필수 구현 (체크리스트)
1. ✅ 커스텀 옵션 UI 정상 노출
2. ✅ 카페24 기본 옵션과 동기화 (선택 상태, 구매 흐름)
3. ✅ 중복 방지 (재선택/반복 클릭 시 오류 없음)
4. ✅ 횟수 제한 (suffix _1, _2 기반)
5. ✅ 금액 계산 정상 (개입수별 추가금액)
6. ✅ 수량/장바구니/바로구매 정상 동작
7. ✅ 활성화 상태 명확히 표시
8. ✅ 외부 설정 JS로 분리 (뱃지, 설명, 가격 등)
9. ✅ 반응형 (PC/MO - 터치 영역 겹침 방지)

### 테스트 상품 설정
- **옵션 방식**: 독립 선택형 (텍스트버튼)
- **옵션값**: 8개 (10개입_1, 10개입_2, 30개입_1, 30개입_2, 50개입_1, 50개입_2, 100개입_1, 100개입_2)
- **금액**: 개입수별로 상이하게 설정

## ⚠️ 주의사항

### 필수 준수
1. **테스트몰에서만 작업** (운영몰 금지)
2. **기본 구매/장바구니 로직 재구현 금지** (기존 로직 활용)
3. **외부 설정 JS는 옵션 UI 구성값 관리 용도만** (구매 로직 대체 금지)
4. **옵션 관련 DOM·이벤트 삭제 금지**
5. **옵션 외 영역 수정 금지** (이미지, 설명, 배너, 구매 버튼 외형 등)

### 구현 우선순위
1. **기능 안정성** (중복 방지, 횟수 제한, 동기화) - 가장 중요
2. **카페24 구조 이해** (기본 옵션과의 연동)
3. **디자인** (Figma 시안 참고하되, 안정성 우선)

### 반응형 체크
- PC: 옵션 영역에서 자연스럽게 노출
- MO: 터치 영역과 구매 버튼 겹침 방지
- 모든 화면 폭에서 텍스트·버튼 깨짐 방지

## 🚨 핵심 체크포인트

### 가장 중요한 부분
1. **옵션 동기화** (#2) - 커스텀 UI와 기본 옵션 연동
2. **중복 방지** (#3) - 반복 클릭 시 오류 없음

### 실패하기 쉬운 부분
- 카페24 기본 DOM 구조 모르고 재구현 → 원래대로 돌림
- 중복 선택 로직 미흡 → QA 탈락
- suffix 개념 놓침 → 횟수 제한 로직 오류
