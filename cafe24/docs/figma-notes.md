### 3. PC 레이아웃 구조

#### 3-1 전체 구성
- **2단 레이아웃**: 좌측 상품 이미지 영역 / 우측 고정폭 구매 패널 (약 350~390px)
- **상품 이미지**: 좌상단, 가로세로 비율 1:1 또는 16:9, 캐러셀(이전/다음 컨트롤)
- **구매 패널**: 우상단 고정 또는 sticky, 스크롤 시 함께 움직임

#### 3-2 옵션 영역 구조
```
┌─────────────────────────────────────┐
│   옵션 선택                          │
├─────────────────────────────────────┤
│  ○ 10개입          24,700원 (25% 할인) │
│     1개: 2,470원                      │
│  ○ 30개입          70,500원 (29% 할인) │   ← 가장 많이 사요
│     1개: 2,350원                      │
│  ○ 50개입          111,500원 (32% 할인)│
│     1개: 2,230원                      │
│  ○ 100개입         196,000원 (41% 할인)│   ← 최대할인
│     1개: 1,960원                      │
└─────────────────────────────────────┘
```

각 행 구조:
- 좌측: 라디오 버튼 (선택/비선택)
- 중앙: 텍스트 (옵션명 + 가격 + 할인율 + 개당가)
- 우측: 뱃지 (조건부 표시)

#### 3-3 선택 후 "선택한 옵션" 요약 박스
```
┌─────────────────────────────────────┐
│  30개입 및 선택                      │  ← 제목 (Bold)
│  떡볶이맛(10개입)*1 + ...          │  ← 선택된 맛 조합 설명
│                                      │
│  수량: [–] 1 [+]       70,500원     │  ← 스테퍼 + 금액
│  [✕ 삭제 버튼]        (우상단)      │
└─────────────────────────────────────┘
```

- 배경: 연살구 `#FFF3EC`
- 테두리: 1px 오렌지 `#FF6B35`
- 패딩: 12~16px

#### 3-4 하단 CTA 영역
```
┌──────────────┬──────────────┬──────────────┐
│  장바구니    │   N 페이     │ 바로 구매    │
│  (아이콘)    │  (연결)      │ (오렌지)     │
└──────────────┴──────────────┴──────────────┘
```
- 3열 동일 높이 (44~48px), 각 버튼이 동일 너비
- "바로 구매": 오렌지 배경 + 흰 텍스트, 가장 강조

### 4. 모바일(MO) 레이아웃 구조

#### 4-1 전체 구성
- **옵션 선택 영역**: **바텀시트(바닥에서 올라오는 모달)** 방식
  - 페이지 스크롤 시 화면 하단에서 등장
  - 반투명 어두운 오버레이(`rgba(0,0,0,0.5)`)
  - 상단에 드래그 핸들(⌄ 아이콘) 포함
  - 닫기 버튼(X) 우상단

#### 4-2 바텀시트 내부 구조
```
┌─────────────────────┐
│  ⌄ (드래그 핸들)     │  ← 상단 영역
├─────────────────────┤
│                      │
│  옵션 선택 (본내용)  │
│  - 라디오 옵션 리스트 │
│  - 선택한 옵션 요약  │
│  - 스테퍼/수량조정   │
│                      │
├─────────────────────┤
│  [선택완료]         │  ← 고정 하단 버튼
│  [장바구니][바로구매]│
└─────────────────────┘
```

#### 4-3 터치 영역 최적화
- 라디오 버튼: 터치 영역 최소 44×44px (실제 버튼은 24~28px, 주위 패딩으로 확장)
- 스테퍼([–], [+]): 정사각형 32~40px, 충분한 간격
- 버튼: 높이 44~48px 이상
- 텍스트: 최소 12px (터치 환경에서 타블렛 고려)

#### 4-4 반응형 중단점
- **모바일 (360~480px)**: 1열, 바텀시트
- **태블릿 (768px 이상)**: PC 2단 레이아웃 (필요 시)

- 형태: 약간의 둥근 모서리 (`border-radius: 6~8px`)

- 폰트 크기: 11~12px, Bold
- 위치: 옵션명 우측 또는 새 줄에 표시

#### 6-2 진행 상황 표시

## 📝 구현 시 주의사항

1. **색상값은 추정치**: 실제 사이트 육안 관찰 기반이므로 미세한 차이 가능. Phase 3 구현 시 브랜드 색상에 맞게 조정 권장.
2. **폰트는 추정**: Noto Sans KR 또는 Pretendard 기반이지만, 실제는 고객사 브랜드 폰트 적용 가능.
3. **간격(padding/margin)**: 정확한 px 값은 스크린샷에서 추출하기 어려우므로, 시각적 일관성을 위해 8px 그리드 시스템 권장 (8, 12, 16, 24, 32 등).
4. **반응형**: 모바일은 바텀시트, PC는 고정 사이드 패널. 태블릿(768px 이상)은 어느 쪽 적용할지 결정 필요.
5. **카페24 기존 DOM**: 기본 옵션 요소(라디오/select)는 `display:none`으로 숨기고, 커스텀 UI로 덮어씀. 이벤트는 숨겨진 요소에 위임.
