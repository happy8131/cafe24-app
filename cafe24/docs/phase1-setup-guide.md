# Phase 1: 환경 준비 및 이해 - 실행 가이드

이 문서는 카페24 로드맵 Phase 1의 1-1(카페24 테스트몰 셋업), 1-2(테스트 상품 설정)을 단계별로 실행할 수 있는 가이드입니다.

---

## ⚠️ 필수 확인사항 (시작 전)

- **계정**: 카페24에 가입되어 있지 않다면 먼저 일반회원 가입 필요
- **테스트몰**: 운영 중인 실제 몰이 있다면 건드리지 말 것 (이 작업은 별도 테스트몰에서만)
- **권한**: 카페24 관리자 계정으로 로그인 필요
- **브라우저**: 크롬, 엣지 등 최신 브라우저 사용 권장 (IE 미지원)

---

## 1-1: 카페24 테스트몰 셋업

### Step 1: 카페24 일반회원가입

1. [카페24 메인 페이지](https://www.cafe24.com) 접속
2. 우상단 "회원가입" 클릭
3. "일반회원" 선택
4. 이메일, 비밀번호, 기본정보 입력 후 가입 완료
5. 이메일 인증 완료 (인증 링크 클릭)

### Step 2: 무료 테스트몰 신청

1. 카페24 로그인
2. 메인 페이지에서 "몰 만들기" 또는 "쇼핑몰 신청" 버튼 클릭
3. 요금제 선택 화면에서 **"3개월 무료"** 또는 **"FREE 플랜"** 선택
4. 아래 항목 입력:
   - **몰 이름**: 예: `test_option_ui` (테스트 목적 명시)
   - **도메인**: 예: `test-option-ui.cafe24.com` (원하는 이름)
   - **업종**: "기타" 선택 (상관없음)
5. 약관 동의 후 신청 완료

### Step 3: 테스트몰 관리자 로그인

1. 메일로 받은 **쇼핑몰 접속 URL** 메모
   - 형식: `https://admin.cafe24.com/shop/{몰아이디}`
2. 브라우저에서 위 주소로 이동 (또는 마이페이지 → "관리자" 클릭)
3. 카페24 계정으로 로그인

### Step 4: 기본 스킨 설정

현재 스킨을 "베이직"으로 변경합니다 (스마트디자인 편집이 쉬운 기본 스킨).

1. 관리자 화면 좌측 메뉴에서 **"디자인" → "디자인 관리"** 클릭
   - 메뉴 경로: 상단 "쇼핑몰" 탭 → 좌측 "디자인" → "디자인 관리"
2. "디자인 보관함" 또는 "스킨" 탭 클릭
3. 오른쪽에 보이는 스킨 목록에서 **"베이직"** 검색 (또는 스크롤)
4. "베이직" 스킨의 **"적용"** 버튼 클릭
5. 확인 창에서 "적용" 재확인

### Step 5: 스마트디자인 편집창 열기

1. 관리자 화면 좌측 메뉴 **"디자인" → "스마트디자인"** 클릭
   - 또는 다이렉트: 관리자 메뉴 → "디자인" → "스마트디자인 편집"
2. 팝업 또는 새 탭에서 스마트디자인 에디터 열림
3. 왼쪽 "페이지 선택" 패널에서 **"상품상세"** 찾기
   - 목록에 없다면 "검색" 아이콘 사용
4. "상품상세" 더블클릭 또는 선택

### Step 6: detail.html 파일 확인

1. 스마트디자인 에디터 중단 영역(주로 코드 보기/에디터 탭)에서 **"HTML/CSS/JS" 탭** 또는 **"소스" 탭** 클릭
2. 아래 영역에 `/product/detail.html` 파일이 보임
   - 또는 전체 파일 목록에서 검색
3. 이 파일을 더블클릭하거나 오른쪽 클릭 → "편집"

**완료 확인**:
- `/product/detail.html` 내용(기본 마크업)이 에디터에 보이면 성공
- HTML 코드가 보이지 않으면 관리자에게 "스마트디자인 편집 권한" 확인

## 1-2: 테스트 상품 설정

### Step 1: 상품 등록 또는 기존 상품 선택

#### 새 상품 등록하기 (권장)

1. 관리자 화면 좌측 메뉴 **"상품" → "상품 관리"** 클릭
2. "상품 등록" 또는 "새 상품" 버튼 클릭
3. 아래 기본정보 입력:
   - **상품명**: `커스텀옵션 테스트 상품` (임의)
   - **판매가**: `10,000` (임의)
   - **상품 이미지**: 아무 이미지나 업로드 (테스트용)
4. 저장 후 계속 진행

#### 기존 상품 선택하기

1. 관리자 화면 **"상품" → "상품 관리"**
2. 기존 상품 목록에서 하나 선택 (이미 옵션이 있다면 삭제하거나 기존 옵션 활용 가능)
3. 해당 상품 "수정" 클릭

### Step 2: 옵션 설정 (가장 중요!)

상품 정보 입력 페이지에서 아래로 스크롤하면 **"옵션" 섹션** 또는 **"상품옵션 설정"** 탭이 있습니다.

#### 옵션 기본 설정

1. "옵션 추가" 또는 "새 옵션" 버튼 클릭
2. **"옵션방식"** 선택 화면:
   - 옵션 타입: **"독립 선택형"** (또는 "독립적 선택") 선택
   - 옵션 입력 방식: **"텍스트버튼"** 선택
3. "다음" 또는 "적용" 클릭

#### 옵션값 입력 (총 8개)

아래 순서대로 옵션값을 입력합니다. 각 행마다 **이름**, **추가금액**을 입력합니다:

| # | 옵션값 이름 | 추가금액 | 비고 |
|---|-----------|--------|------|
| 1 | `10개입_1` | `2,000` | 10개 첫 번째 |
| 2 | `10개입_2` | `2,000` | 10개 두 번째 |
| 3 | `30개입_1` | `5,000` | 30개 첫 번째 |
| 4 | `30개입_2` | `5,000` | 30개 두 번째 |
| 5 | `50개입_1` | `8,000` | 50개 첫 번째 |
| 6 | `50개입_2` | `8,000` | 50개 두 번째 |
| 7 | `100개입_1` | `15,000` | 100개 첫 번째 |
| 8 | `100개입_2` | `15,000` | 100개 두 번째 |

**입력 방법**:
1. 각 행의 "옵션값 이름" 입력칸에 위 이름 입력
2. "추가금액" 또는 "가격" 입력칸에 해당 금액 입력
3. "+" 또는 "추가" 버튼 클릭해 다음 행 추가
4. 8개 모두 입력 후 "적용" 또는 "저장" 클릭

**주의**: `_1`, `_2` suffix는 **정확하게** 입력해야 함. 나중에 JavaScript 로직에서 이를 기반으로 횟수 제한을 계산합니다.

### Step 3: 옵션 저장 및 상품 등록 완료

1. 옵션 입력 완료 후 전체 상품 정보 페이지의 **"저장"** 또는 **"상품등록"** 버튼 클릭
2. "등록 완료" 또는 "수정 완료" 메시지 확인

### Step 4: 상품 상세페이지에서 옵션 확인

1. 상품이 등록되면 테스트몰 **스토어프론트** 접속:
   - 관리자 화면에서 "스토어 보기" 또는 주소: `https://{몰도메인}.cafe24.com`
2. 검색창에서 위에서 등록한 상품명 검색
3. 상품 상세페이지 진입
4. 상품 설명 아래에 **옵션 선택 드롭다운** 또는 **텍스트버튼**이 보이는지 확인
   - 8개 옵션값(`10개입_1`, `10개입_2`, ... , `100개입_2`)이 모두 노출되면 성공

**만약 옵션이 보이지 않는 경우**:
- 관리자에 다시 로그인해 상품 "수정" 클릭
- 옵션이 제대로 저장되었는지 확인
- 브라우저 새로고침 (Ctrl+F5 또는 Cmd+Shift+R)

## 📞 문제 해결

### 카페24 로그인이 안 됨
- 비밀번호 재설정 시도: [카페24 비밀번호 찾기](https://www.cafe24.com/login/)
- 2차 인증이 활성화되어 있다면 앱 또는 이메일 인증 필요

### 스마트디자인 편집 권한이 없음
- 카페24 관리자에게 "스마트디자인 편집 권한" 요청
- 또는 계정 설정 → "권한 관리"에서 확인

### 옵션이 상품 상세페이지에 보이지 않음
- 상품이 "판매중" 상태인지 확인 (상태 > "판매중"으로 설정)
- 스킨 템플릿이 옵션을 지원하는지 확인
- 브라우저 캐시 삭제 후 새로고침

### 옵션값이 저장되지 않음
- 각 행의 "추가금액" 칸이 비어있는지 확인 (모두 입력 필요)
- suffix (`_1`, `_2`)에 띄어쓰기나 특수문자가 없는지 확인
- 저장 버튼을 다시 클릭

### suffix (_1, _2)는 왜 필요한가?
- 나중에 Phase 4에서 JavaScript 로직이 이 suffix를 기반으로 **횟수 제한**을 계산합니다
- 예: `30개입_1`, `30개입_2` → 30개입은 최대 2회 추가 가능
- 정확하게 입력하지 않으면 로직이 제대로 작동하지 않습니다
