# Phase 6: 테스트 및 QA 체크리스트

## 개요

Phase 4~5에서 구현한 커스텀 옵션 UI를 체계적으로 검증하기 위한 체크리스트입니다. 각 항목에는 테스트 방법, 기대 결과, 현재 상태가 기록되어 있습니다.

### #4: 횟수 제한 테스트 ✅

**테스트 방법:**
같은 그룹의 두 카드를 모두 선택한 후, 같은 그룹 카드 재선택 시도

예: "10개입_1" → "10개입_2" → 그룹 가득 참 → "10개입" 그룹의 다른 카드 클릭 불가 또는 경고

**기대 결과:**
- 그룹당 최대 개수 도달 시: "10개입은(는) 최대 2개까지만 추가 가능합니다" 경고 토스트
- 선택 불가, "선택한 상품"에 추가 안 됨
- 미선택 카드들이 disabled 스타일(흐려짐, opacity 0.6)로 표시

**현재 상태:** ✅ 완료 (사용자 확인함)

**회귀 테스트:**
- Phase 6 수정 후 재확인

### #5: 금액 계산 정상 확인 ✅

**테스트 방법:**
다양한 카드 조합 선택 후 합계 금액 확인

예:
- "10개입_1" 선택 → 합계: +2,000원 ✓
- "30개입_1" 추가 선택 → 합계: +7,000원 ✓
- "10개입_1" 수량을 2로 변경 → 합계: +9,000원 (2×2,000 + 5,000) ✓

**기대 결과:**
- 합계 = Σ(option_price × quantity)
- 실시간 갱신 (클릭/수량변경 시 즉시 반영)

**현재 상태:** ✅ 완료 (사용자 확인함)

**회귀 테스트:**
- Phase 6 수정 후 다양한 조합으로 재확인

### #7: 활성화 상태 표시 확인 ✅

**테스트 방법:**
선택된 카드와 비선택 카드의 시각적 구분 확인

**기대 결과:**
- 선택된 카드: 테두리 주황색(#FF6B35, 2px), 배경색 옅은 주황색(#FFF3EC), 우측 상단에 흰 원형 체크마크(✓) 표시
- 미선택 카드: 기본 테두리 회색(#EEE, 1px), 배경색 흰색
- 호버 상태: 배경색이 짙어짐
- 비활성화(횟수 초과): opacity 0.6, 포인터 이벤트 차단

**현재 상태:** ✅ 완료 (사용자 확인함)

**회귀 테스트:**

## 🟡 부분 검증 (보강 완료, 실 DOM 대기)

### #2: 커스텀 ↔ 기본 옵션 동기화 ⚠️

**테스트 방법 (로컬, 제한적):**
- preview.html은 `.xans-product-option` 요소가 없어 실제 동기화 불가능
- 콘솔에서 에러 없이 `syncToDefaultOption()` 함수 실행되는지 확인 (콘솔 에러 없음이면 OK)

**기대 결과 (실 스토어에서):**
- 커스텀 UI 카드 클릭 → 기본 옵션 select/버튼 선택 상태 동시에 변경
- 기본 옵션 선택 → 커스텀 UI 카드도 선택 상태로 업데이트

**현재 상태:** ⚠️ 부분 검증
- ✅ 로직 보강 완료: `syncToDefaultOption()` select/버튼 둘 다 지원
- ✅ 이벤트 등록 완료: `attachEventListeners()`에서 버튼 클릭 감지
- ❌ 실제 동작 미검증 (실 카페24 DOM 미확보)

**재검증 방법 (실 스토어 준비 후):**
1. 카페24 테스트몰에서 상품 상세페이지 로드
2. 커스텀 UI 카드 클릭 → 기본 옵션(select 또는 버튼)도 함께 선택되는지 확인
3. 기본 옵션 직접 변경 → 커스텀 UI 카드 상태가 동시에 변경되는지 확인
4. 콘솔 에러 없는지 확인

**주의:**
- `cafe24_roadmap.md` 첫 번째에 "옵션 방식: 독립 선택형 (텍스트버튼)"이라 명시되어 있음
- `docs/phase2-dom-analysis.md` 초안도 버튼형을 유력하게 봄
- 실제 카페24 환경에서 구조가 다르면 `docs/phase2-dom-analysis.md` 체크리스트로 재분석 필요

### #6: 수량/장바구니/바로구매 정상 동작 ⚠️

**테스트 방법 (로컬, 부분):**
- ✅ 수량 UI: "+"/"-" 버튼으로 수량 조절, 합계 금액 즉시 갱신 (검증됨)
- ✅ 삭제 기능: 선택 목록에서 삭제 버튼으로 항목 제거 (검증됨)
- ❌ 장바구니 추가: 실제 폼 제출 불가능 (preview.html에는 submit 버튼 없음)
- ❌ 바로구매: 실제 폼 제출 불가능

**기대 결과 (실 스토어에서):**
- 선택 항목들이 숨겨진 input 필드(`option_hidden_1`, `option_quantity_1` 등)에 올바르게 저장됨
- 장바구니 추가 → 선택한 옵션들이 장바구니에 포함됨
- 바로구매 → 선택한 옵션들이 주문 폼에 포함됨

**현재 상태:** ⚠️ 부분 검증
- ✅ 수량 조절 UI: 정상 동작 (선택 목록의 +/- 버튼)
- ✅ 삭제 기능: 정상 동작
- ✅ 합계 계산: 수량 반영하여 정상 계산
- ⚠️ 폼 제출: 실 스토어 필요 (로컬에서는 제출 폼이 없음)

**재검증 방법 (실 스토어 준비 후):**
1. 카페24 상품 상세페이지에서 옵션 여러 개 선택 (수량도 조절)
2. 장바구니 추가 버튼 클릭
3. 장바구니 페이지에서 옵션과 수량이 올바르게 저장되었는지 확인
4. 바로구매도 동일하게 검증
