# Phase 6-4: 화면 녹화 계획 및 시나리오

## 개요

로드맵 Phase 6-4에서 요구하는 4개 화면 녹화(#2 동기화, #3 중복방지, #4 횟수제한, #7 활성화 상태)를 실 스토어 준비 후 촬영할 때 참고하기 위한 상세 시나리오입니다.

---

## 📹 녹화 1: #2 동기화 (5~10초)

### 목표
커스텀 UI와 기본 옵션이 **실시간으로 동기화**됨을 보여줍니다.

### 시나리오

**씬 1: 커스텀 UI 카드 클릭 → 기본 옵션도 변경**

1. 카페24 상품 상세페이지 로드 상태로 시작
2. 기본 옵션 드롭다운(또는 버튼) 초기값 화면에 보이도록 조정
3. 커스텀 UI에서 "30개입_1" 카드 클릭
4. **동시에 확인:**
   - 커스텀 UI: 카드에 주황색 테두리 + 체크마크 표시
   - 기본 옵션: select 값이 "30개입_1"로 변경되거나, 버튼이 선택 상태로 변경
5. 화면에 두 UI 모두 보이도록 스크롤 조정하여 촬영

**씬 2: 기본 옵션 직접 선택 → 커스텀 UI도 변경 (선택사항)**

1. 기본 옵션 드롭다운에서 "50개입_2" 선택
2. 커스텀 UI에서 해당 카드도 자동으로 선택 상태로 변경되는지 확인
3. 두 UI가 동기화됨을 시각적으로 확인

### 기대 효과
- 개발자가 둘 다 지원했고, 실제로 작동함을 증명
- 사용자가 어느 쪽 UI를 사용해도 동일한 효과

## 📹 녹화 2: #3 중복 방지 (5~8초)

### 목표
같은 옵션을 두 번 클릭했을 때 **중복 추가가 방지**됨을 명확히 보여줍니다.

### 시나리오

1. 커스텀 UI "10개입_1" 카드 클릭
2. 화면 변화 캡처:
   - 카드에 체크마크 표시
   - "선택한 상품" 섹션에 "10개입" 항목 추가
   - 성공 토스트 "10개입 선택됨" 표시 (짧게 떠났다가 사라짐)
3. 즉시 **같은 카드를 다시 클릭**
4. 화면 변화 캡처:
   - 경고 토스트 "이미 선택한 옵션입니다" 표시
   - "선택한 상품"에는 여전히 1개만 있음 (중복 추가 안 됨)

### 기대 효과
- 중복 클릭이 안전하게 처리됨
- 실수로 같은 옵션을 여러 번 눌러도 오류 없음

### 촬영 팁
- 토스트 메시지가 짧으므로, 느린 모션이나 슬로우 플레이백 고려
- 또는 토스트 메시지가 보일 때까지 기다렸다가 녹화 시작

## 📹 녹화 3: #4 횟수 제한 (8~12초)

### 목표
그룹당 최대 개수 제한이 정확하게 작동함을 보여줍니다.

### 시나리오

1. 커스텀 UI에서 "10개입" 그룹의 첫 번째 카드 "10개입_1" 클릭
2. 선택됨 확인 (테두리 주황색, 체크마크)
3. 같은 그룹의 두 번째 카드 "10개입_2" 클릭
4. 선택됨 확인 (테두리 주황색, 체크마크)
5. "선택한 상품" 섹션을 스크롤하여 두 항목이 모두 추가되었음을 보여줌
6. **그룹이 가득 찼으므로**, 다른 그룹의 카드를 클릭하여 비교:
   - "30개입_1" 클릭 → 정상 선택됨 (다른 그룹이므로)
   - "선택한 상품"에 세 번째 항목 추가
7. **다시 "10개입" 그룹 카드를 클릭 시도:**
   - 경고 토스트 "10개입은(는) 최대 2개까지만 추가 가능합니다" 표시
   - 항목이 추가되지 않음

### 카메라 각도
- 카드들이 흐려지는 모습(disabled 스타일)이 보이도록 하면 좋음
- "선택한 상품" 섹션도 함께 보여야 항목 개수 증가가 명확

### 기대 효과
- 같은 그룹의 옵션을 제한된 개수만 선택 가능
- 초과 선택 시도 시 경고 알림
- 다른 그룹은 독립적으로 선택 가능

## 📹 녹화 4: #7 활성화 상태 표시 (8~10초)

### 목표
선택/미선택/비활성화 상태가 시각적으로 명확하게 구분됨을 보여줍니다.

### 시나리오

**씬 1: 선택 상태 표시**

1. 여러 그룹에서 각각 옵션 선택:
   - "10개입_1" 클릭
   - "30개입_1" 클릭
   - "50개입_1" 클릭
2. 화면에서 3개 카드의 스타일을 비교:
   - **선택됨**: 주황색(#FF6B35) 테두리(2px), 옅은 주황색(#FFF3EC) 배경, 흰 원형 체크마크(✓)
   - **미선택**: 회색(#EEE) 테두리(1px), 흰 배경

**씬 2: 비활성화 상태 표시**

1. "10개입" 그룹의 두 번째 카드도 클릭하여 그룹을 가득 채움
2. "10개입" 그룹의 미선택 카드들이 **disabled 상태**로 표시:
   - opacity 0.6으로 흐려짐
   - 배경색이 회색(#F0F0F0)으로 변함
   - 포인터가 'not-allowed'로 변함 (호버 시 클릭 불가 시각화)

**씬 3: 호버 상태**

1. 선택되지 않은 카드에 마우스 호버
2. 배경색이 밝아짐(#F5F5F5)을 확인
3. 선택된 카드에도 호버 → 배경색 변화 없음 (이미 선택된 상태이므로)

### 카메라 각도
- 여러 카드가 동시에 보이는 와이드 뷰
- 비활성화된 카드들이 흐려지는 모습이 명확하게 보이도록

### 기대 효과
- 사용자가 현재 UI 상태를 직관적으로 파악 가능
- 선택된 것, 미선택된 것, 선택할 수 없는 것이 시각적으로 구분됨

## 🎬 촬영 기술

### 도구

#### 데스크탑 (권장)
1. **Chrome 개발자도구 "Device Mode"**
   - F12 → Device Mode 활성화 → 모바일 기기 선택
   - 반응형을 보여줄 수 있음
   - OBS Studio, ScreenFlow(Mac) 등으로 녹화

2. **데스크탑 스크린 레코더**
   - Windows 11: Win + Alt + R (게임 바)
   - Mac: Cmd + Shift + 5
   - 또는 OBS Studio (무료, 고급 편집 가능)

#### 모바일 (실제 장치)
- **iOS**: 제어 센터 → 화면 기록 시작
- **Android**: 설정 → 시스템 도구 → 스크린 레코드

### 포맷

| 구분 | 형식 | 해상도 | 길이 | 용도 |
|------|------|--------|------|------|
| GIF | .gif | 800×600 이상 | 10초 이내 | 카톡, 이메일, 문서 첨부 |
| 동영상 | .mp4 (H264) | 1280×720 이상 | 1분 이내 | YouTube, 클라우드 스토리지 |
| 슬로우 모션 | .mp4 (0.5x) | - | 20초 이내 | 토스트 메시지 등 빠른 요소 강조 |

### 편집 (선택사항)

- **GIF 제작**: EZGIF.com, LiceCAP (온라인/오프라인)
- **영상 편집**: OBS Studio (자막, 배경음악), Adobe Premiere (고급), 또는 소박하게 그냥 MP4

### 배포

- **파일 크기 최소화**
  - 화면 영역만 크롭 (전체 모니터 제외)
  - GIF는 8bit 색상, 프레임 속도 15fps로 압축
  - MP4는 1280×720, H264, 5Mbps 비트레이트

- **업로드**
  - 이메일: 10MB 이하 (GIF 권장)
  - GitHub/Notion: 직접 업로드 또는 YouTube 링크
  - Google Drive/Dropbox: 링크 공유
